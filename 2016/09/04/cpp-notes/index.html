一个派生类对象如果是public继承的，那么可以被当做一个基类对象来使用

note 2:
In the static binding, the decision about which function to use to resolve an overload is made at compile time.
静态绑定表示的是调用哪个函数在编译时候已经决定。

If a member function is declared to be virtual, dynamic binding is used. The decision about which function to use to resolve an overload is made at run time, if it cannot be determined at compile time:
Virtualness is inherited.
It is override but not overload.
A run-time decision is needed only when an object is accessed through a pointer or reference to a base class.
1 如果声明为virtual，则为动态绑定，决定调用哪个函数是在运行时决定的，编译时不能决定。
2 虚函数是被继承的。
3 是覆盖不是重载。
4 动态绑定一般是当通过指针访问基类成员时。

note 3:
A method is declared abstract by specifying that it is virtual and by supplying = 0 in the interface in place of an implementation.
An abstract method has no meaningful definition and is thus always defined in the derived class.
A class that has at least one abstract method is called an abstract class Because the behavior of an abstract class is not completely defined, abstract classes can never be instantiated.
When a derived class fails to override an abstract method with an implementation, it remains abstract, and the compiler reports an error if an attempt to instantiate the abstract derived class is made.
抽象方法：纯需函数，抽象类实际上就是画一个框架，它的所有孩子都拥有一样的框架，只是每个人的有不一样
1 抽象函数是在函数前加virtual，后面加上=0
2 抽象函数没有实际意义的定义，总是在子类里被定义
3 至少有一个纯需函数的类叫做抽象类
4 因为没有被完整的定义，所以抽象类不能被实例化
5 当一个子类没有写抽象方法的实现时，这个子类仍然是抽象类，如果实例化仍然会出错

Exception Handling
C++ exception handling centers around the use of the try keyword and the catch keyword
The C++ Standard Library provides and uses a standard exception class hierarchy.
Head file: #include <stdexcept>

In C++, a variable of any data type can be thrown as an exception using the throw statement.

A function can transfer execution to a catch statement that exists in another function. This usually happens when a function (that throws an exception) is called from within a try block of another function.

C++ provides a mechanism to catch exceptions of all possible types.
